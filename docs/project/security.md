# 🔐 セキュリティ

## 認証と認可

※ クライアント側での認証管理は重要ですが、リソースを保護するためにサーバー側でも堅牢なセキュリティ対策を実装することが同様に重要です。クライアント側の認証はユーザー体験を向上させ、サーバー側のセキュリティを補完するものです。

リソースの保護は以下の2つの要素で構成されます。

### 認証（Authentication）

認証とはユーザーの身元を確認するプロセスです。SPA（シングルページアプリケーション）では、JSON Web Token（[JWT](https://jwt.io/)）によるユーザー認証が一般的です。ユーザーがログインまたは登録すると、アプリ内に保存されるトークンが発行されます。その後、認証が必要なリクエストごとにヘッダーまたは Cookie にトークンを付与して送信し、ユーザーのアイデンティティとアクセス権限を検証します。

最も安全な方法はトークンをアプリの状態として保持することですが、ユーザーがページをリフレッシュするとトークンがリセットされ認証状態が失われます。

そのため、トークンは Cookie か `localStorage/sessionStorage` に保存する必要があります。

#### `localStorage` vs Cookie でのトークン保存

`localStorage` にトークンを保存することは、XSS（クロスサイトスクリプティング）の脆弱性によってトークンが盗まれるリスクがあります。

`HttpOnly` 属性を設定した Cookie にトークンを保存することで、クライアント側の JavaScript からアクセスできなくなりセキュリティが向上します。このサンプルアプリでは Cookie 管理に js-cookie を使用しています。実際の API では `HttpOnly` 属性が設定されており、クライアント側からは Cookie の内容にアクセスできないことを前提としています。

さらに、XSS 攻撃からアプリ全体を守るために、表示するすべてのユーザー入力をサニタイズすることが重要です。

[HTML サニタイズのコード例](../apps/react-vite/src/components/ui/md-preview/md-preview.tsx)

セキュリティリスクの全一覧は [OWASP](https://owasp.org/www-project-top-10-client-side-security-risks/) を参照してください。

#### ユーザーデータの扱い

ユーザー情報はアプリ全体からアクセスできるグローバルな状態として管理すべきです。`react-query` を使用している場合は、[react-query-auth](https://github.com/alan2207/react-query-auth) ライブラリが設定を提供するだけで状態管理をすべて担ってくれます。それ以外の場合は、React context + フック、またはサードパーティの状態管理ライブラリを活用してください。

[認証設定のコード例](../apps/react-vite/src/lib/auth.tsx)

ユーザーオブジェクトが存在する場合、アプリはそのユーザーが認証済みとみなします。

### 認可（Authorization）

認可とは、ユーザーがアプリ内の特定リソースへのアクセス権限を持っているかを確認するプロセスです。

#### RBAC（ロールベースアクセス制御）

[認可設定のコード例](../apps/react-vite/src/lib/authorization.tsx)

ロールベースの認可モデルでは、特定のロールを定義してそれに権限を関連付けることでリソースへのアクセスを制御します。例えば `USER` と `ADMIN` のようなロールに異なるアクセス権限を設定できます。管理者には全機能へのアクセスを許可し、一般ユーザーには特定の機能を制限するといった制御が可能です。

[RBAC のコード例](../apps/react-vite/src/features/discussions/components/create-discussion.tsx)

#### PBAC（パーミッションベースアクセス制御）

RBAC は構造化された認可の方法を提供しますが、より細粒度の制御が必要な場合もあります。PBAC は「リソースのオーナーのみが操作を実行できる」といった特定の条件に基づいて権限を細かく調整できる柔軟なソリューションです。例えばコメントの場合、そのコメントの作者のみが削除できるといった制御を実現します。

RBAC 保護には `RBAC` コンポーネントに許可ロールを渡す方法を使います。より厳格な保護が必要な場合はポリシーチェックを渡すことができます。

[PBAC のコード例](../apps/react-vite/src/features/comments/components/comments-list.tsx)
