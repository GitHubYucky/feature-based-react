# ⚠️ エラーハンドリング

### API エラー

インターセプターを実装してエラーを適切に処理してください。このインターセプターを使って、エラーをユーザーに通知するトーストを表示したり、未認証ユーザーをログアウトさせたり、トークン更新リクエストを送信してセキュアかつシームレスな操作を維持したりできます。

[API エラー通知のコード例](../apps/react-vite/src/lib/api-client.ts)

### アプリ内エラー

React の Error Boundary を使用して、アプリの特定箇所で発生したエラーを処理してください。アプリ全体に1つだけ Error Boundary を置くのではなく、複数の箇所に配置することを検討してください。こうすることで、エラーが発生してもアプリ全体の機能を停止させずにローカルで封じ込めることができ、よりスムーズなユーザー体験を実現できます。

[Error Boundary のコード例](../apps/react-vite/src/app/routes/app/discussions/discussion.tsx)

### エラー追跡

本番環境で発生するエラーは追跡する必要があります。独自実装も可能ですが、[Sentry](https://sentry.io/) のようなツールを使う方が効果的です。アプリをクラッシュさせる問題を報告してくれるほか、どのプラットフォームやブラウザで発生したかも確認できます。ソースコードのどこでエラーが起きたかを確認するために、ソースマップを Sentry にアップロードするようにしてください。
