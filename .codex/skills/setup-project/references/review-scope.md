# Review Scope

`setup-project` で優先して確認する対象。

## 先に見る場所

1. `docs/ideas/`
- 初期要求、壁打ちメモ、ユーザーの方向性を確認する。

2. `docs/`
- 既存の恒久文書、テンプレート由来文書、過去の分析メモを見分ける。

3. `src/`
- 実際の構成、feature 単位の責務、技術スタックを確認する。

4. `.claude/`
- 既存エージェント運用資産、コマンド、skills、テンプレートを確認する。

## 整理するときの観点

- 現在の実装を表す記述か
- 目標アーキテクチャを表す記述か
- テンプレートまたは過去のガイドか
- 既存 skill や command と重複していないか
- 後続の設計文書作成に必要な前提が足りているか

## 最低限まとめること

- どの要求が `docs/ideas/` から来ているか
- どの docs が現在の実装に追随しているか
- どの docs が古いか、または対象外か
- 次の文書化を進めるうえで不足している前提
