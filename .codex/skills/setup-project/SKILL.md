# Setup Project

## Use this skill when

- ユーザーが初回セットアップを依頼したとき
- リポジトリ全体を把握して、次の作業の入口を整理したいとき
- `docs`、`.claude`、`src` をまたいで現状確認したいとき
- 必要な設計文書や共通ルールの不足を洗い出したいとき

## Goal

- リポジトリの現行構造と管理対象を把握する
- 実装と docs のずれを整理する
- 初回に必要な設計文書や共通ルールの不足を見つける
- 次に着手すべき実装または文書化の入口を明確にする

## Workflow

1. ルート構成、`src`、`docs`、`.claude` を確認する。
2. 既存 docs を読み、現行実装とのずれや未整備箇所を整理する。
3. このリポジトリで意味のある管理対象を分類する。
4. 既存の設計文書や運用ルールが足りない場合は、何を追加すべきか明確にする。
5. 必要なら分析ドキュメントや共通ルールを作成または更新する。
6. 次に進むべき実装または文書化の優先アクションをまとめる。

## Rules

- 広いリファクタ前提で進めず、まず現状把握を優先する。
- テンプレート由来の資料よりも、現行の `src` と実際に使われている docs を優先して判断する。
- `docs`、`.claude`、`src` を別物として扱わず、相互の整合性を意識する。
- 不一致を見つけた場合は、現状実装、目標像、テンプレート由来の記述を区別して記録する。
- 変更提案は、次に安全に着手できる粒度まで具体化する。
