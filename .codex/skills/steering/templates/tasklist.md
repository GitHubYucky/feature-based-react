# タスクリスト

## 🚨 タスク完全完了の原則

**このファイルの全タスクが完了するまで作業を継続すること**

### 必須ルール
- **全てのタスクを `[x]` にすること**
- tasklist を正式記録として逐次更新すること
- テスト先行を基本にすること
- 実装ファイル 1 つに対して、原則として対応テストファイル 1 つを作ること
- 未完了タスク（`[ ]`）を残したまま作業を終了しない

### 禁止事項
- 「時間の都合により後回し」にする
- 実装だけ進めて tasklist を更新しない
- 複数の実装を 1 つの曖昧なテストでまとめる

### スキップが許可されるケース
- 実装方針変更でタスクが不要になった
- アーキテクチャ変更で別方式に置き換わった
- 依存関係変更でタスクが成立しなくなった

スキップ時は必ず理由を明記:

```markdown
- [x] ~~タスク名~~（実装方針変更により不要: 具体的な技術的理由）
```
