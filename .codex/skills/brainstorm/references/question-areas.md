# 壁打ち観点

`initial-requirements.md` に落とすため、必要に応じて次の観点を使って整理する。

## 1. 目的

- 何を作りたいか
- なぜ作りたいか
- どんな課題を解決したいか

## 2. 想定ユーザー

- 誰が使うか
- どんな状況で使うか
- その人にとって何が価値か

## 3. 最初に必要な機能

- 最初のリリースで外せない機能
- その中でも最優先の 1 機能
- 成功判定に直結する機能

## 4. 次に欲しい機能

- あると便利だが初回では必須でない機能
- 反応を見て追加判断できる機能

## 5. 制約

- 技術的制約
- スケジュール制約
- 触りたくない範囲
- 避けたい大きな変更

## 6. 出力の形

最後は次の形でまとめる。

```md
# {プロジェクト名} Initial Requirements

最終更新: {日付}

## 目的

## 想定ユーザー

## まず欲しい機能

## 次に欲しい機能

## 制約

## 次の依頼例
```
