# Doc Review

## Use this skill when

- ユーザーが docs のレビューを依頼したとき
- 設計文書や運用文書の質を確認したいとき

## Workflow

1. 対象 docs を読む。
2. 現行コードや関連 docs と照合する。
3. 次の観点で評価する。
- 完全性
- 明確性
- 一貫性
- 実装可能性
4. 問題点を優先度順にまとめる。
5. 必要なら改善案を提案する。
